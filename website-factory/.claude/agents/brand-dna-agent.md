# brand-dna-agent

Stage 7 executor. Extracts client brand DNA into `brand-dna.json` for downstream
consumption by Stage 9 (hero generation) and Stage 10.1 (template overlay build).

This agent extracts the universal brand DNA fields that every niche template needs.
The structure decisions (which sections render, in what order) come from the niche
template itself (scaffolded by Module 2D), not from a per-client `archetype`. Brand
DNA stays comprehensive (palette, typography, motif, certifications, founder, contact,
hours, services, areas), plus these two niche-agnostic fields:

- `theme_mode: "light" | "dark"`, single mode decision (no toggle ever, default light)
- `voice_register: "commercial" | "family" | "premium"`, copy tone hint for Stage 6

`shape_motif` is still extracted because the per-client BackgroundPattern component in
the niche template selects from its motif SVG library (universal motifs + any niche-
specific additions) based on it.

## Inputs

- `clients/[Client Name]/Pipeline Data/research/research.json`
- `clients/[Client Name]/[Client Name] Assets/logo/logo.{svg,png}`
- `clients/[Client Name]/[Client Name] Assets/project-images/` (optional)
- `clients/[Client Name]/Pipeline Data/copy/copy-deck.md`
- `clients/[Client Name]/Pipeline Data/brand-resonance/resonance.json` (OPTIONAL, present only when Stage 7.5 has been built and run; absence is fine)

## Process

### Step 0, Read accumulated lessons (REQUIRED)

Before any other step, read these two files if they exist and apply every rule
listed as an override to this agent spec:

1. `.claude/lessons/by-agent/brand-dna-agent.md`, universal corrections that apply to every run of this agent
2. `clients/[Client Name]/Pipeline Data/lessons/notes.md`, corrections specific to this client only

Lessons take precedence over the default behaviour in this spec.

### Step 1, Read the design-synthesis skill and impeccable references

Read `.claude/skills/design-synthesis/SKILL.md` end-to-end before starting.
The skill defines the five-pass synthesis process, palette synthesis rules,
shape mode picker, and confidence scoring methodology.

Also read before proceeding:
- `.claude/skills/impeccable/skill/reference/brand.md`, font reflex-reject list and aesthetic lane test.
- `.claude/skills/impeccable/skill/reference/color-and-contrast.md`, color quality calibration.

### Step 2, Verify inputs

Confirm:
- Logo file exists at `clients/[Client Name]/[Client Name] Assets/logo/`. If missing, halt with a clear error pointing to `MANUAL-DROP-NEEDED.md`.
- `research.json` is valid and contains GBP data. If missing, halt and request Stage 2 be re-run.
- `copy-deck.md` has been generated by Stage 6.

### Step 3, Run the five passes

Execute the synthesis passes from the skill in order:

1. Logo analysis (extract dominant colors + brightness signature + motif description)
2. Palette synthesis (primary, primary_dark, accent, accent_light, neutral, ink)
3. Typography selection (heading + body fonts with Google Fonts URL fragments)
4. Hero mood + tagline
5. Motif extraction (shape_motif derived from logo motif_description keywords)

`shape_motif` valid values: `polygon | triangle | wave | arc | dot-grid | hexagon | chevron | diamond | cross-hatch | mountain | shingle | blueprint-grid | topographic`. Default to `polygon` if extraction is ambiguous.

Compute aggregate confidence after Pass 5.

### Step 3.5, Theme mode decision (NEW for template-approach)

Decide `theme_mode: "light" | "dark"` from these signals (in priority order):
1. If `resonance.json` exists and contains `theme_mode_recommendation`, use that.
2. Else if logo brightness signature shows a dark logo on light background (logo darker than 0.5 luminance), default to `light`.
3. Else if logo brightness signature shows a light logo on dark background (logo brighter than 0.5 luminance), default to `dark`.
4. Else default to `light`.

Record the decision rationale in `extraction-report.md`.

### Step 3.6, Voice register decision (NEW for template-approach)

Decide `voice_register: "commercial" | "family" | "premium"` from niche-aware
signals. The criteria below are the contractor / home-services default;
non-contractor niches override via
`templates/{active-niche-slug}/niche-playbook/vocabulary.json` and
`trust-signals.json` (Module 2D writes a `segmentVocabulary` block per
niche listing premium / family / commercial indicators).

The niche playbook's `vocabulary.json` defines a `segmentVocabulary` block
that maps each register to the niche-appropriate signals (e.g. residential
vs commercial vs mixed for contractor; year-round vs seasonal for
hospitality; bespoke vs production for fashion). For each register, the
playbook lists the service keywords, segment labels, ticket-size cutoffs,
and certification flags Stage 7 should check.

If the playbook does not supply a segment vocabulary for the active niche,
default to `family` and write a warning to `extraction-report.md` so the
student knows to populate the playbook segment vocabulary for the niche
before running another client through.

Tiebreaker: default to `family` if no clear signal.

### Step 3.7, Per-client accent stop derivation (REQUIRED)

After Step 3 produces `palette.accent`, run:

```bash
python3 tools/derive-accent-stops.py --client "[Client Name]" --write
```

This patches `brand-dna.json` in place to add `palette.accent_light`,
`palette.accent_mid`, `palette.accent_dark` derived from the actual accent via
luminance-aware HSL shifts.

### Step 3.8, Corner overlay decision (REQUIRED)

Write `corner_overlay = {motif, color, opacity}` so the `CornerOverlay`
component carries the per-client motif silhouette + accent tint into
every section that anchors corner overlays. See `09-build.md` Rule 58
and `brand-dna-agent.md` Rule 5.

Per-client variance for corner overlays is BOTH shape AND color, and
the motif decision is INDEPENDENT from `shape_motif`. Pick the corner
silhouette that fits the brand's character separately from the
BackgroundPattern motif.

Decision logic:

1. `motif` is an explicit per-client pick from the 10 available
   silhouettes (one SVG asset per option at
   `templates/{niche-slug}/src/assets/corner-overlays/{motif}.svg`). Do
   NOT default to `shape_motif`. Use the per-motif character guide in
   `lessons/by-agent/brand-dna-agent.md` Rule 5 to pick:
   - `rectangle` for clean / no-frills / solid-block brands
   - `triangle` for classic-roofing / pitched / traditional
   - `polygon` for modern / multi-faceted / contemporary
   - `hexagon` for structured / premium / engineering-led
   - `chevron` for momentum / progressive / fast
   - `diamond` for premium / luxury / value-stacked
   - `arc` for soft / approachable / family
   - `wave` for organic / playful / relaxed
   - `mountain` for outdoors / heritage / rugged
   - `shingle` for trade-craft-visible / material-led
2. `color` defaults to `palette.accent`. Override only when a secondary
   palette colour reads better at low opacity (rare).
3. `opacity` defaults to `0.08`. Range `0.04 - 0.18`. Use the low end
   for light section backgrounds, the high end for darker.

Record the rationale in `extraction-report.md`. If Stage 7 does not
write this block, `tools/build-from-template.py` `compose_brand_dna()`
fills the default fallback (motif = shape_motif, color = palette.accent,
opacity = 0.08), so the build never fails on a missing field.

### Step 4, Compose brand-dna.json

Assemble the output object. Every required field must be populated, no nulls,
no empty strings, no placeholders.

Pull from `research.json` for:
- contact (phone, email, address, service_areas, address_url for Google Maps)
- business_hours (open, close, tz extracted from GBP hours)
- trust (license, ratings, review counts, accreditation status, years_in_business)
- founder (name, first_name, title, story)
- founding_year (scraped from old-site copy or GBP "Established" text, required by Stage 13 proposal)
- certifications (the niche playbook at `templates/{active-niche}/niche-playbook/trust-signals.json` defines which certs/affiliations to look for and how to populate them)
- segment (segments come from the niche playbook's segment vocabulary, e.g. residential/commercial/mixed for contractor; year-round/seasonal for hospitality)
- region (mapped from address)
- region_marketing (e.g. "Greater Houston" for Stage 13; derive from city/state)
- state_full (e.g. "Texas" for Stage 13; map STATE_CODE→STATE_FULL)
- insurance_focus (extracted from copy-deck voice)

**Founder name parsing (NEW for template-approach Stage 13):** when `research.json` provides `ownerName` as a single string (e.g. `"{{REFERENCE_OWNER_NAME}}"`), split on first whitespace. Write both:
- `founder.name` = full name
- `founder.first_name` = first token only
The proposal uses `first_name` in conversational copy ("Brandon will call within 5 minutes").

**State code → State full mapping (NEW for template-approach Stage 13):** use the standard US 2-letter to full-name lookup. Common cases: TX→Texas, CA→California, FL→Florida, MN→Minnesota, MO→Missouri, AZ→Arizona, MI→Michigan, GA→Georgia, NY→New York, NC→North Carolina, etc. The proposal substitutes both `{{STATE_CODE}}` and `{{STATE_FULL}}` from these.

**Region marketing label (NEW for template-approach Stage 13):** derive from primary city + state with this priority:
1. If `research.json` already has `region_marketing` (a Stage 7.5 brand-resonance signal), use it.
2. Else common metros, TX/Houston→"Greater Houston", TX/Dallas→"Dallas-Fort Worth Metroplex", AZ/Phoenix→"Greater Phoenix", FL/Miami→"South Florida", etc.
3. Else fall back to `"Greater {city}"`.

Pull from synthesis passes for:
- palette (primary, primary_dark, accent, accent_light, neutral, ink)
- typography (heading, body, heading_url, body_url)
- shape_motif (drives template's BackgroundPattern selection)
- shape_mode (sharp | softened, drives base radius)
- hero (eyebrow / H1 / tagline / mood)
- taglines, logo metadata

Compute deterministically:
- `theme_mode` from Step 3.5
- `voice_register` from Step 3.6

**Removed from prior schema:** `archetype`, `component_variance`,
`archetype_confidence`, `theme_toggle`, `has_owner_cutout`,
`founder.cutout_path`. Stage 8 (Claude design system) is gone, so
`brand_board_slug` is also removed.

### Step 5, Validate against schema

Two schemas govern the brand-dna output:

1. `references/schemas/brand-dna.schema.json` documents the universal shape +
   field semantics; validate the JSON object you write against it with any
   JSON Schema validator (Node: `ajv`, Python: `jsonschema`).
2. `templates/{niche-slug}/src/config/brand-dna.example.js` is the
   canonical JS object shape every component reads. Stage 10.1
   (`tools/build-from-template.py`) merges your `brand-dna.json` output into
   this shape and the prebuild hook (`scripts/validate-brand-dna.mjs`) halts
   the build if any required field is missing or contains a surviving
   `__REQUIRED__SOMETHING__` sentinel.

Populate as many fields as possible from research + resonance + intake.
Surviving sentinels surface in Stage 10.1 as build halts with precise
dotted paths.

If validation fails: print the failing field and reason, halt the pipeline,
do NOT write the file.

If validation passes: write to
`clients/[Client Name]/Pipeline Data/brand/brand-dna.json` with 2-space indent.

### Step 6, Write extraction report

Write `clients/[Client Name]/Pipeline Data/brand/extraction-report.md` with:

- Summary table: region, shape_mode, shape_motif, theme_mode, voice_register, palette swatches, typography
- Per-pass confidence scores
- Aggregate confidence
- Theme mode decision rationale
- Voice register decision rationale
- Any gaps flagged for human review

### Step 7, Update pipeline state and log

Append to `clients/[Client Name]/Pipeline Data/logs/build-log.md`:

```
## Stage 7, Brand DNA extraction
Status: complete
Aggregate confidence: 0.XX
Palette: {primary} / {accent}
Typography: {heading} / {body}
Shape motif: {shape_motif}
Theme mode: {theme_mode}
Voice register: {voice_register}
Region: {region}
Output: clients/[Client Name]/Pipeline Data/brand/brand-dna.json
```

Update `clients/[Client Name]/Pipeline Data/logs/pipeline-state.json`:
```json
{ "stage_7": "complete", "stage_7_confidence": 0.XX }
```

If aggregate confidence < 0.7, mark `stage_7: pending_review` instead and halt.
The user runs `/approve-brand-dna` to proceed.

## Output gate

Stage 9 (hero image) and Stage 10.1 (template overlay build) cannot proceed
until brand-dna.json validates AND pipeline-state.json shows `stage_7: complete`.

## Failure handling

| Failure | Action |
|---------|--------|
| Logo file missing | Halt, write MANUAL-DROP-NEEDED.md, do not retry |
| research.json malformed | Halt, request re-run of Stage 2 |
| Schema validation fails | Halt, log failing field, do not retry until input is corrected |
| Aggregate confidence < 0.7 | Halt with `pending_review`, generate report, wait for user |
| WCAG AA contrast fails after shifts | Generate an alternative palette using a contrast-safe accent that meets WCAG AA against both light and dark text. Log the chosen value in extraction-report.md. |
