# Niche Template Scaffold Blueprint

The file-by-file contract for what a **completed** niche template at
`website-factory/templates/{niche-slug}/` must contain after Module 2D's
12 phases finish. This is the **product** document; the 12-phase
**process** lives in `.claude/agents/14-template-builder.md`.

## Two-layer architecture

Every niche template is the composition of two layers:

1. **Universal blueprint** at `website-factory/references/factory-blueprint/`.
   The deterministic plumbing `website-factory/tools/generate-factory.py`
   Phases 1-3 stamps this into `templates/{niche-slug}/` with `{{slot}}`
   tokens substituted from the niche design tokens. Same shape for every
   niche.
2. **Per-niche layer** generated by Claude-driven Phases 6-10. Section
   components, route pages, niche playbook (JSON + markdown contracts),
   per-niche SOPs + agents + checklists. Different per niche.

The canonical brand-dna shape contract at
`website-factory/references/brand-dna.shape.js` lives outside both layers:
Phase 3 stamps it into `src/config/brand-dna.example.js` (the runtime
contract) + a sentinel-laden `src/config/brand-dna.js` (the per-client
fill target Stage 10.1 populates).

## File-by-file contract

### `package.json`

Phase 2 stamps from `references/factory-blueprint/package.json.template`.
Substitutes the package `name` to `{niche-slug}-template`. Deps are
universal (react, react-dom, react-router-dom, framer-motion, tailwindcss,
vite, eslint + plugins).

### `vite.config.js`, `vercel.json`, `eslint.config.js`, `postcss.config.js`, `.gitignore`

Copied verbatim from the blueprint. No niche substitution; same content
across every niche.

### `tailwind.config.js`

Phase 2 stamps from `tailwind.config.js.template`. Substitutes:
- `{{TYPOGRAPHY_HEADING}}` / `{{TYPOGRAPHY_BODY}}` to the niche's font families
- `{{MOTION_DURATION_MS}}` / `{{MOTION_STAGGER_MS}}` to the niche's motion preset
- 10 `{{PALETTE_*_RGB}}` slots to the niche's palette as CSS-rgb triples

### `index.html`

Phase 2 stamps from `index.html.template`. Substitutes `{{META_TITLE}}`,
`{{META_DESCRIPTION}}`, `{{GOOGLE_FONTS_URL}}`, `{{JSON_LD}}`,
`{{NICHE_SLUG}}`, `{{THEME_MODE}}`. The runtime per-client values come
from Stage 10.1's `inject-theme.mjs` overlay, not this template.

### `scripts/inject-theme.mjs` + `scripts/validate-brand-dna.mjs`

Universal prebuild hooks. `inject-theme.mjs` rewrites the `:root` palette
block in `src/index.css` per the active client's `brand-dna.json`.
`validate-brand-dna.mjs` walks the canonical shape contract at
`src/config/brand-dna.example.js` and halts the build if any field is
missing or has the wrong type. Both copied verbatim from the blueprint.

### `src/App.jsx`

Phase 2 stamps from `App.jsx.template`, **leaving `{{PAGE_IMPORTS}}` and
`{{ROUTES}}` intact** for Phase 7 to fill. Phase 7 emits:
- One `import { PageName } from './pages/PageName.jsx';` per route in
  `09-sitemap.json`
- One `<Route path="..." element={<PageName />} />` per route
- A `<Route path="*" element={<NotFoundPage />} />` 404 fallback as the
  last entry

### `src/index.css`

Phase 2 stamps from `index.css.template`. Carries:
- `:root` block with niche palette CSS variables (token-substituted)
- Google Fonts `@import` matching the niche typography preset
- Foundation utilities (modular type scale, 8pt spacing, layered shadows,
  premium easings, motion utilities, `prefers-reduced-motion: reduce`
  block, `tabular-nums` utility, smart-quote audit comment)

### `src/config/brand-dna.example.js`

Phase 3 stamps `references/brand-dna.shape.js` into this file, renaming
the export from `brandDNAShape` to `brandDNA`. This is the canonical
contract: every component reads from `brandDNA.X.Y.Z` paths that resolve
in THIS file. Gate 1's Layer B grep validates JSX accesses against it.

### `src/config/brand-dna.js`

Phase 3 writes a sentinel-laden copy of the canonical shape (every leaf
value is `__REQUIRED__FIELD_NAME__`). Stage 10.1 overlays per-client
values, filling the sentinels. The Phase 1 prebuild hook
(`validate-brand-dna.mjs` without `--shape-only`) halts the per-client
build if any sentinel survives.

### `src/components/{SectionName}.jsx` (one per wireframe section)

Phase 6 (Claude-driven) generates one component per section in the niche
wireframe. Each component:
- Imports `brandDNA` from `../config/brand-dna`
- Reads ONLY canonical paths (Gate 1 enforces; off-shape access fails)
- Implements the wireframe's layout variant (split-screen / asymmetric /
  centered / full-bleed / etc.)
- Uses Tailwind classes referencing the niche token CSS variables
- Honours `prefers-reduced-motion`
- Contains no em-dashes + no AI-vocab blocklist terms
- Self-audits against impeccable's 5-dimension audit framework

### `src/pages/{PageName}.jsx` + `NotFoundPage.jsx` (one per sitemap route + 1)

Phase 7 (Claude-driven). Each page is a thin shell:
- Imports section components from `../components/`
- Composes them in the order the wireframe specifies
- Carries no business logic, only composition
- Uses semantic HTML (`<main>`, `<section>`, `<nav>`, `<aside>`, `<footer>`)
- Heading hierarchy starts at one `<h1>` per page
- Mobile-first responsive (375px first, then `md:` / `lg:` breakpoints)
- No inline styles
- `NotFoundPage.jsx` is a semantic 404 with a primary CTA back to home

### `niche-playbook/{name}.json` (10 required files)

Phase 8 (Claude-driven). Each file:
- Validates against its schema at `website-factory/references/niche-playbook/schemas/{name}.schema.json`
- Pulls values from specific niche-research artifacts (`09-template-spec.md`, `02-customer-voice.md`, `05-trust-signals.md`, etc.) — see the Phase 8 table in `14-template-builder.md` for the per-file source
- No nulls / empty strings; counts derive from research, not "feels right" defaults
- String values use the niche's end-customer vocabulary (no AI-vocab blocklist terms)

Required: `copy-locks.json`, `trust-signals.json`, `process.json`,
`vocabulary.json`, `motion-preset.json`, `theme.json`,
`hero-mood-mapping.json`, `photo-manifest.json`, `asset-patterns.json`,
`proposal-pages.json`. Gate 6 enforces every one.

### `niche-playbook/{name}.md` (6+ markdown contracts)

Phase 8 also writes markdown contracts derived from `website-factory/references/niche-playbook/contracts/*.contract.md`:

- `copywriting.md`, niche voice + tone guidance
- `cro-rules.md`, conversion mechanics derived from top-of-niche analysis
- `design-vocabulary.md`, per-site one-liners + layout vocabulary catalogue + typography pairings + motion idioms
- `hero-composition.md`, complete per-niche hero image prompt template Stage 9 reads (composition spec + subject + mood baseline + region defaults + style ladder)
- `quantified-trust-templates.md`, niche-specific trust statement templates
- `copy-blocklist-additions.md`, niche-specific terms to add to the universal copy blocklist
- `sop-overrides/00-master.md` and other sop-overrides as the contract surface dictates

Gate 6 enforces `hero-composition.md`, `copywriting.md`, `design-vocabulary.md`, `cro-rules.md` as required.

### `niche-playbook/trust-badges/`

Curated niche-specific SVG badge files OR a `MANUAL-DROP-NEEDED.md` listing each badge with its issuer URL when SVGs aren't readily available. The student curates the SVGs manually after Module 2D completes.

### `.claude/sops/NN-{stage-name}.sop.md` (per Phase 9 matrix)

Phase 9 (Claude-driven). One SOP per pipeline stage that benefits from
niche-specific knowledge (see `14-template-builder.md` Phase 9 matrix).
Each SOP:
- Has every `{{slot}}` filled from the niche playbook + wireframe (Gate 6 enforces; no surviving `{{`)
- Is at least 600 chars (Gate 6 enforces; aim for 1000-2000)
- Substantively tailored, not a stub

### `.claude/agents/{NN-stage-name}.md` (per Phase 9 matrix)

Phase 9. One agent per tailored pipeline stage:
- Every `{{slot}}` filled (Gate 6 enforces)
- At least 600 chars (Gate 6 enforces; aim for 2000-4000)
- Invokes the relevant universal skill(s) from `website-factory/.claude/skills/` (Gate 6 enforces at least one skill invocation across the entire agent set)
- Has required sections: `## Role`, `## Prerequisites`, `## Steps`, `## Pass gate` or `## Failure handling`, `## What this agent never does`

### `.claude/checklists/sop-compliance.md`

Phase 10 (Claude-driven). Per-section + per-stage compliance checks derived from the niche wireframe. The sop-qa-agent merges this with the universal `website-factory/.claude/checklists/sop-compliance.universal.md` at runtime. No surviving `{{slot}}` placeholders (Gate 6 enforces).

### `.claude/checklists/design-fidelity.md`

Phase 10. Per-region SSIM weights + thresholds. Region weights derived from conversion-critical sections; thresholds from `09-template-spec.md` fidelity table. The design-fidelity-qa-agent merges this with the universal layer at runtime. No surviving `{{slot}}` placeholders.

### `MANIFEST.json`

Phase 9 of `generate-factory.py` (the deterministic orchestrator). Lists every generated file with size + checksum so downstream tooling can verify the niche template's integrity. Gate 6 requires it exists.

## Per-client variance (locked dimensions)

Every niche template is filled with per-client values at Stage 10.1 by
overlaying `clients/[Client Name]/Pipeline Data/brand/brand-dna.json`
onto the sentinel-laden `src/config/brand-dna.js`. Stage 10.1's
`build-from-template.py` halts if any sentinel survives the overlay
(the Phase 1 prebuild hook enforces this in `validate-brand-dna.mjs`).

The niche template + niche playbook + universal SOPs + universal checklists
together define the **invariant** — every client in this niche ships
the same site **shape**, just with their own brand, copy, photos, and
content slotted in.

## Source inspiration

The reference pool the niche template was scaffolded from lives at
`research/02-niche-research/{niche-slug}/templates/raw/{site-slug}/`
(8-12 captured top-of-niche sites with desktop + mobile screenshots,
DOM, CSS, fonts, colors). Each niche template MANIFEST references the
pool + the winner pick (`templates/pick.json`) for traceability.

## Validation

Module 2D Phase 11 runs the 6-gate validator (`tools/validate-niche-template.py`)
against the completed niche template. On Gate 6 tailoring failures, Phase 11
loops back into the relevant Claude-driven phase up to 3 times before
writing `GENERATION-FAILED.md` and halting. See
`.claude/agents/14-template-builder.md` Phase 11 for the loop-back protocol.
